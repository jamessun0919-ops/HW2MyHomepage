Restaurant Helper

Click to Start Code Repository

Provides functionality closely aligned with everyday needs, reducing the daily difficulty of deciding where to eat. Compared to similar apps on the market, this project integrates with an LLM to process a richer set of user input conditions — fixed factors such as health profile (age, gender), health management goals (muscle gain, fat loss, blood sugar control), and dietary preferences (vegetarian type / spice tolerance), combined with variable factors like the day's mood and dining motivation, further constrained by budget and distance — to generate an optimal search prompt that surfaces the dining choice best suited to the user's conditions.

For the restaurant recommendation mechanism, AI assistance was used to design a system that balances precise recommendations with a relaxed-search fallback. In dense dining areas (universities, office districts, hospitals, malls, food streets) where the first round already offers plenty of options, the system provides recommendations that precisely match the user's conditions. In relatively remote areas (scenic spots, rural areas, residential districts), a second-round mechanism relaxes the tier matching and reasonably expands the search distance, ensuring comprehensive and stable restaurant results.

In terms of user experience, this project uses a lightweight web design with no cumbersome registration or login, and no intrusive ads. Personalization settings and history are stored only in the browser (clearable with one click); users can quickly rate their satisfaction with the previous recommendation and choose whether to avoid the previously recommended restaurant for variety.
